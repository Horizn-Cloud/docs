---
title: "Get article components"
openapi: "GET /articles/{id}/components"
description: "Get all components or a specific component type"
---

Returns article components (body, SEO metadata, schemas, social snippets, etc.). Optionally filter by component type.

**Scope required:** `articles:read`

## Path parameters

<ParamField path="id" type="string" required>UUID of the article.</ParamField>

## Optional path

Append `/{type}` to get a single component type:

```
GET /articles/{id}/components/SeoMetadata
```

See [Component types](/api-reference/reference/component-types) for all available types.
