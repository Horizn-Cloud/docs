<code>articles:write</code>

The `articles:write` scope allows you to create articles from completed IdeaClouds and trigger article generation. Articles can be created in simple mode (fully automated) or interactive mode (you choose the questions and options before generation begins).

## Endpoints

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/public/articles` | Create a single article |
| `POST` | `/public/articles/batch` | Create articles in batch |
| `POST` | `/public/articles/{id}/generate` | Generate an interactive-mode article |

---

## Create article

```
POST /public/articles
```

Creates a new article from a completed IdeaCloud. In `simple` mode, generation starts immediately. In `interactive` mode, the response includes the available inputs you need to provide before calling the [Generate](/scopes/articles-write#generate-interactive-mode-article) endpoint.

### Request body

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `ideacloud_id` | string (UUID) | Yes | The IdeaCloud to generate the article from |
| `article_type` | string | Yes | One of: `basic`, `qa`, `expert`, `modellanding`, `comparison`, `salesevent` |
| `selected_question_ids` | array of UUIDs | No | Question IDs from the IdeaCloud to focus on (min 1 if provided) |
| `brand_voice_id` | string | No | Brand voice ID to use for tone and style |
| `title` | string | No | Custom article title (max 500 chars) |
| `auto_compliance` | boolean | No | Automatically run compliance check after generation. Default `false` |
| `auto_content_tools` | boolean | No | Automatically generate content tools after generation. Default `false` |
| `mode` | string | No | `simple` (default) or `interactive` |

```json
{
  "data": {
    "id": null,
    "status": "awaiting_input",
    "ideacloud_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "article_type": "qa",
    "required_inputs": {
      "selected_question_ids": {
        "type": "array",
        "required": true,
        "options": [
          {
            "id": "q1a2b3c4-d5e6-7890-abcd-ef1234567890",
            "question": "What is the starting price of the 2026 Honda Civic?"
          },
          {
            "id": "q2b3c4d5-e6f7-8901-bcde-f12345678901",
            "question": "How does the Corolla's fuel economy compare to the Civic?"
          }
        ]
      },
      "brand_voice_id": {
        "type": "string",
        "required": false
      },
      "title": {
        "type": "string",
        "required": false
      }
    }
  }
}
```

<Note>
In interactive mode, no article is created yet. Use the `required_inputs` to build your follow-up request to the [Generate](/scopes/articles-write#generate-interactive-mode-article) endpoint.
</Note>

### Status codes

| Code | Description |
| --- | --- |
| `200` | Interactive mode — awaiting input |
| `202` | Simple mode — article created and generation started |
| `400` | Validation error (missing or invalid fields) |
| `401` | Missing or invalid API key |
| `403` | API key does not have `articles:write` scope |
| `404` | IdeaCloud not found |
| `429` | Rate limit exceeded |
| `500` | Internal server error |

## Create articles in batch

```
POST /public/articles/batch
```

Creates multiple articles in a single request. Each item is processed independently. Only `simple` mode is supported in batch requests.

### Request body

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `items` | array | Yes | Array of article objects (1-10 items) |

Each item accepts the same fields as the single [Create article](/scopes/articles-write#create-article) endpoint.

### Example

<CodeGroup>
```bash cURL
curl -X POST https://public-api.hrizn.io/public/articles/batch \
  -H "X-API-Key: hzk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {
        "ideacloud_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
        "article_type": "comparison",
        "auto_compliance": true

        "article_type": "basic",
```

## Generate interactive-mode article

```
POST /public/articles/{id}/generate
```

Triggers generation for an article that was created in interactive mode. You must provide the required inputs returned by the [Create article](/scopes/articles-write#create-article) endpoint.

### Path parameters

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `id` | string (UUID) | Yes | Article ID (from the interactive-mode create response) |

### Request body

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `selected_question_ids` | array of UUIDs | Yes | Question IDs to focus on (min 1) |
| `brand_voice_id` | string | No | Brand voice ID |
| `title` | string | No | Custom article title (max 500 chars) |

### Example

<CodeGroup>
```bash cURL
curl -X POST https://public-api.hrizn.io/public/articles/d4e5f6a7-b8c9-0123-def0-456789abcdef/generate \
  -H "X-API-Key: hzk_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "selected_question_ids": [
      "q1a2b3c4-d5e6-7890-abcd-ef1234567890",
      "q2b3c4d5-e6f7-8901-bcde-f12345678901"
    ],
    "brand_voice_id": "bv-premium-001",
    "title": "Honda Civic vs Toyota Corolla: Which Should You Buy?"
  }'
```

## Error response — Missing scope

If your API key does not include `articles:write`, any request to these endpoints returns:

```json
{
  "error": {
    "code": "forbidden",
    "message": "This API key requires one of the following scopes: articles:write",
    "details": {
      "required_scopes": ["articles:write"]
    }
  }
}
```
